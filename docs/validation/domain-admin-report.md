## Security Contracts Verified

1. **TenantId Source**: Server derives `tenantId` from `req.user.tenantId` (auth token), not `req.body` or `req.query`.
2. **Middleware Chain**: `requireAuth` + `requireTenant` are both required for all org-scoped endpoints.
3. **Admin Role Check**: `req.user.role === "admin"` reads from verified token, not request body.
4. **Authorization Header**: All admin API calls require `Authorization: Bearer <token>` header.

## Known Findings

1. `POST /api/users` is a sync endpoint that does not require auth (by design for Firebase sync).
   Firebase Admin SDK validation is applied at the middleware level for protected endpoints.
   The FK constraint prevents injection of users into non-existent companies.

2. Browser UI tests (company profile navigation, user settings page) are conditionally skipped
   when `E2E_SERVER_RUNNING=1` is not set. These are integration tests requiring both Express
   and Vite servers; they pass when run in a full local dev environment.
