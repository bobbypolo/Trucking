# Knowledge Base

This directory contains accumulated project knowledge that persists across sessions.

## Files

| File | Purpose | Updated By |
|------|---------|------------|
| `lessons.md` | Issues encountered and how they were resolved | `/learn` skill |

## How to Use

### Adding Lessons

When something unexpected happens during development:

1. Resolve the issue
2. Run `/learn`
3. The Librarian agent will capture the lesson in `lessons.md`

### Lesson Format

Each lesson includes:
- **Date and Title**: For quick scanning
- **Tags**: For searchability (#api, #docker, #testing, etc.)
- **Issue**: What went wrong
- **Root Cause**: Why it happened
- **Resolution**: How it was fixed
- **Prevention**: How to avoid it next time

## Best Practices

- Keep lessons concise (under 200 words)
- Use descriptive tags for filtering
- Focus on prevention, not just fixes
- Link to relevant files when helpful
